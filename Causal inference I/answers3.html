Preliminaries

This document contains the 3rd practical for the MSc course Causal Inference I. In Lecture 3 we have seen the basic terminology and usage of directed acyclic graphs (DAGs). In this practical we will be gaining some hands-on experience with these graphical objects. The learning goals for this practical are:

  1. To practice with DAG terminology;
  2. To practice the derivation of testable implications of a given DAG;
  3. To practice with the identification of causal effects from a given DAG.

The programming language of choice is R. In the remainder code will be contained in blocks and in-text references to functions, function arguments, and objects will be denoted as foo. The code in code-blocks can be directly copied by pressing the copy to clipboard button in the upper-right corner. Code may be directly pasted in your R console.

Exercises

Our first 3 exercises relate to the consecutive learning goals listed above. In Exercise 1 we will practice with the familial language used to understand DAGs and their structure. Exercise 2 we will practice with the derivation of testable implications from a DAG. We will see that we may want to amend or adapt a given DAG after evaluation of the testable implications. Exercise 4 is an additional (or take-home) exercise. Exercise 1: DAGs as a family gathering

We have seen in the Lecture that DAGs are simply graphical objects. Graphical objects are simply collections of nodes (or vertices) and edges (or connections). What makes a graphical object a DAG is when all edges are unidirectional arrows (the directed part) and that, when tracing the directions indicated by those arrows, we never can get back to a node we started from (the acyclic part). Such graphs are used to intuitively represent causal beliefs through present arrows (encoding direct causal influence) as well as missing arrows (encoding the absence of direct causal influence). In order to harness their intuitive power, and in order to understand the ever-growing research literature on DAGs, you need to be comfortable with their terminology. In the lecture we have seen that DAGs draw upon familial notions to express causal structure and influence. We will gain some further practice with that here.

Imagine you are a sports medicine researcher specialized in injuries. You amass your knowledge and experience in a DAG that expresses your causal beliefs. You are especially interested in the effect of Warm-Up Exercises (WUE) on Injuries (I). Furthermore, your causal beliefs include the effects of:

  • Coach (C);
  • Team Motivation (TM);
  • Previous Injuries (PI);
  • Contact Sports (CS);
  • Fitness Level (FL);
  • Pre-Game Proprioception (a neuronal measure of sense of self-movement, force, and body position) (PGP);
  • Intra-Game Proprioception (IGP);
  • Neuromuscular Fatigue (NMF).

We are going to use the dagitty function to draw our DAG. A statement like C [pos="-4.000,-3.000"] specifies the node C as well as its (relative) location. A statement like C -> { FL TM } specifies that there is a directed connection from C to FL as well as a directed connection from C to TM. In the code below we explicitly indicate I as our outcome and WUE as our exposure of interest. The following specification then represents and plots your causal beliefs:

## Specify the DAG
g.postulated <- dagitty('dag {
  C   [pos="-4.000,-3.000"]
  CS  [pos="-4.000,3.000"]
  FL  [pos="1.000,-3.000"]
  I   [outcome,pos="2.500,3.000"]
  IGP [pos="-1.000,3.000"]
  NMF [pos="2.500,-1.000"]
  PGP [pos="-1.000,-2.000"]
  PI  [pos="-4.000,1.000"]
  TM  [pos="-4.000,-1.000"]
  WUE [exposure,pos="-1.000,0.000"]
  C   -> { FL TM }
  CS  -> { IGP PI }
  FL  -> { NMF PGP }
  IGP -> I
  NMF -> { I IGP }
  PGP -> WUE
  TM  -> { IGP PI }
  WUE -> IGP
  }
')

## Plotting DAG
plot(g.postulated)

Let us now try to understand this DAG with the use of familial nomenclature as seen during the lecture:

  2. Who are the ancestors of I?
  3. Who are the children of NMF?
  4. Name all descendants of T.
  5. Draw all directed paths from C to I.
  6. Is IGP a collider in the triplet TM, IGP, NMF?

Exercise 2: Art thou independent?

We will continue with our setting from Exercise 1. After we have drawn a DAG and have understood its structure we may want to evaluate if the DAG is consistent with (the information contained in) data. This is where we evaluate the testable implications of a DAG and see if they hold up in empirical practice. As we know by now, the concept of d-separation can be used to assess the restrictions (in terms of conditional and unconditional independencies) that must hold in any dataset generated by a mechanism that has a given DAG as the underlying data generating process. Let us review some of these implications for the DAG given in Exercise 1.

  1. Which set of variables d-separates C and CS?
  2. Which set of variables d-separates WUE and I?
  3. List as many implied conditional independencies as you can.


  1. The empty set. This is because, for example, the collider PI blocks the flow of information between C and CS. Hence, in this case, not conditioning on a collider would leave the path C - TM - PI - CS blocked. This goes for all possible paths between C and CS. Thus,
C⊥ ⁣ ⁣ ⁣⊥CSC \perp\!\!\!\perp CSC⊥⊥CS

  1. It is clear that we must condition on IGP to block the flow of information between WUE and I. In doing so, we open up a path through NMF, as IGP is a collider for WUE and NMF. We then must additionally condition on NMF to render WUE and I independent. Hence: WUE⊥ ⁣ ⁣ ⁣⊥I ∣ IGP,NMF.WUE \perp\!\!\!\perp I \,|\, IGP, NMF.WUE⊥⊥I∣IGP,NMF.

We have collected empirical data on the variables of interest from n = 500 athletes. The variables all pertain to continuous measurements and have been standardized. These data are available in the data object from the sportsData.RData file on BrightSpace. This means that we can test whether our DAG is consistent with the data at hand. Let us first load and visually inspect the data:

## Load and inspect data
load("sportsData.RData")
ggpairs(data)+ theme_bw()

Our variables are all approximately normally-distributed. Which implies that they are jointly normally-distributed. In this normal case, conditional independence is equivalent to zero partial correlation. We can use regression techniques to obtain partial correlations. For example, the partial correlation between X and Y given Z can be obtained by regression both X and Y on Z and evaluating the correlation between the respective residuals. This is what the localTests function will do for us internally. It uses the d-separation criterion to list all implied conditional independencies (as obtained through impliedConditionalIndependencies) and applies a formal test of zero (partial) correlation for each of these identifies independencies. Let us perform this empirical evaluation. We invoke the local tests of interest on our postulated DAG and apply a Holm-Bonferroni correction on the p-values. We then plot the local tests that have a low corrected p-value:

## Perform local tests
r <- localTests(g.postulated, data)

## Adjust using Holm-Bonferroni correction
## Retain only those tests with adjusted p-value < .05
r <- r[p.adjust(r$p.value) < 0.05, ]

## Plot conditional independencies inconsistent with data
r <- r[order(r$p.value), ]
par(mar = c(4, 8.5, 1.5, 0.5))
plotLocalTestResults(r, bty = "n", xlim = c(-0.4, 0.4), ylim = c(0.1, 3.1),
                     axis.pars = list(las = 1, lty = 0))

  1. Why would we perform a multiple testing correction?
  2. What are your conclusions based on the output?
  3. Say that we assume that the inconsistencies noticed in 11 are not due to measurement error nor due to unmeasured (latent) variables. How should you then amend your DAG given these conclusions?


  1. As we have many testable implications (64) we would be capitalizing on chance without corrections. We use Holm-Bonferroni as it makes no assumption of independence between the consecutive tests.
  2. Looking at the output we see that we have 3 implied conditional independencies whose empirical partial correlation is clearly nonzero. Hence TM ⊥ WUE | C, TM ⊥ WUE | FL, and TM ⊥ WUE | PGP are found to be inconsistent with the collected data.
  3. All inconsistencies pertain to TM and WUE. The inconsistencies imply that TM cannot be made independent from WUE. Hence, a directed link from TM to WUE should be included. Indeed, you could also include an edge from WUE to TM, which would also be consistent with the concurrent conditional independencies and the data. From a substantive viewpoint, however, an edge from TM to WUE seems more logical. This stresses the importance of the incorporation of expert/substantive knowledge in causal reasoning.

Additional (take-home) exercise: Predictor without a cause

During the lecture we learned that causal DAGs can be understood as structural equation models. Let us have a look at a simple structural equation system: X:=ϵXY:=X+ϵYZ:=Y+ϵZ, \begin{aligned} X &:= \epsilon_{X} \\ Y &:= X + \epsilon_{Y} \\ Z &:= Y + \epsilon_{Z}, \end{aligned} XYZ​:=ϵX​:=X+ϵY​:=Y+ϵZ​,​ with ϵX ∼ 𝒩(0,1), ϵY ∼ 𝒩(0,1), and ϵZ ∼ 𝒩(0,0.1). In the causal structural model literature, the symbol := implies causal assignment. Hence, X is measured with random error, Y is causally influenced by X, and Z is causally influenced by Y. Let us generate and plot some data according to this model.

## Generate and plot data
set.seed(1234)
n <- 1000
x <- rnorm(n, 0, 1)
y <- x + rnorm(n, 0, 1)
z <- y + rnorm(n, 0, 0.1)
DAT <- data.frame(x,y,z)
xyplot <- ggplot(DAT, aes(x = x, y = y)) + geom_point()  
zyplot <- ggplot(DAT, aes(x = z, y = y)) + geom_point()  
grid.arrange(xyplot, zyplot, ncol = 2)

  1. Which, from a regression standpoint, would be a better predictor for Y: X or Z?
  2. It is clear, from our simulation, that Z does not have a causal effect on Y. What does this small example convey?


  1. Clearly, Y has a weaker association with X than with Z. Hence, Z is therefore more relevant when it comes to prediction (from a regression perspective). This is immediate from the figures. We can also conclude this from regressing Y on X and regressing Y on Z:

Recap and look ahead

DAGs encode qualitative causal beliefs in graphical objects. A DAG gives intuitive access to the testable implications underlying causal beliefs. Moreover, DAGs are very useful for identifying confounding variables that require conditioning when estimating causal effects. The topic of estimation of causal effects will be the focus of the next lecture and practical.

Note that an absence of violations of the testable implications of a DAG does not equate to proof that the DAG is correct. In fact, there may be equivalent DAGs: different DAGs (with differing causal implications) that carry the same testable implications. It is important to realize that it is the belief in a given causal syntax that makes a DAG (and subsequent analyzes) causal. These and other issues are beyond the scope of this course. If you really like DAGs, you will learn more about them in the elective course Causal Inference II.
